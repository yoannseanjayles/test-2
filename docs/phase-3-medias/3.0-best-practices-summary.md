# Phase 3.0 — Best Practices Summary : Médias d'un e-commerce premium

> **Statut** : 🟡 En attente de validation HITL
> **Équipe** : Directeur Artistique, UX/UI Designer, Motion Designer, Architecte Front-end (formats/perf)
> **S'appuie sur** : positionnement D-008, specs Phase 2 (hero lifestyle D-022, galerie PDP D-024, vignettes éditoriales D-037).

## Rôle des médias dans notre contexte

Sur un site premium, **l'image EST la preuve du positionnement** : la curation se voit avant de se lire. Nos médias doivent tenir trois promesses simultanées : qualité perçue (DA cohérente), information (juger taille/matière — D-024) et performance (LCP < 2,5 s — D-009).

## 1. Styles photographiques — les 4 registres du site

| Registre | Usage | Caractéristiques |
|---|---|---|
| **Packshot studio** | Listing, galerie PDP (vue 1), Google Shopping | Fond neutre uniforme (blanc cassé/greige plutôt que blanc pur — signature premium), lumière douce, ombre portée légère, produit net à 100 % |
| **Détail matière** | Galerie PDP | Macro sur coutures, grain du cuir, boucles ; lumière latérale qui révèle la texture |
| **Lifestyle animal** | Hero, cartes univers, PDP (portée/à l'échelle), éditorial | Animaux réels en situation, intérieurs chaleureux (bois, lin, tons naturels), lumière naturelle dorée ; l'animal est digne, jamais déguisé/ridiculisé (D-008) |
| **Flat-lay** | Collections, guides, e-mails, réseaux sociaux | Vue zénithale composée, accessoires groupés par usage, fond texturé neutre |

**Règle de cohérence** : la direction artistique est **documentée et verrouillée** (références approuvées, palette, lumière, retouche) — jamais dépendante de « l'œil » d'un photographe ou d'une génération isolée. C'est l'objet de la bibliothèque de prompts (3.2).

## 2. Visuels attendus par surface (issus des specs Phase 2)

- **Hero accueil** : lifestyle cinématique avec espace négatif pour le texte (H1 + CTA) — zone de texte à prévoir dans la composition.
- **Cartes univers (×3)** : portraits animal cadrés serrés, fond cohérent entre les 3.
- **PDP** : 5–8 visuels/produit — packshot multi-angles, macro matière, **portée à l'échelle avec gabarit en légende** (D-024), dimensions annotées.
- **Éditorial** : couvertures d'articles (situation + animal), schémas de mesure (guide des tailles).
- **Marque** : atelier/matières/équipe pour `/notre-histoire`.
- **Transverses** : icônes (réassurance, piliers, UI), illustrations (états vides, 404, e-mails), Open Graph, e-mails transactionnels et marketing.

## 3. Formats, ratios, technique

| Usage | Ratio | Taille source | Notes |
|---|---|---|---|
| Packshot PDP/listing | **1:1** | 2000×2000 px | Zoom PDP ; < 2 Mo source, AVIF/WebP en diffusion |
| Hero desktop | 16:9 → 21:9 | 2560 px large | Recadrage art-directed mobile (pas un simple crop) : version 4:5 dédiée |
| Cartes univers/catégorie | 4:5 | 1200×1500 px | Cohérence verticale |
| Éditorial (couverture) | 3:2 | 1600×1067 px | + variante 1:1 pour cartes |
| Open Graph | 1.91:1 | 1200×630 px | Texte lisible à petite taille |
| E-mail | largeur 600 px | ×2 pour retina | Poids strict |
| Réseaux sociaux | 1:1 et 4:5 | 1080/1350 px | 4:5 prioritaire (feed mobile) |

- **Diffusion** : AVIF avec fallback WebP, `srcset` responsive, lazy-loading sous le pli, hero préchargé (spec 2.1 Accueil).
- **Google Shopping** : packshot fond neutre, ≥ 1000 px, sans texte ni logo incrusté.
- Icônes : SVG uniquement (stroke cohérent, défini en Phase 4).

## 4. Spécificités génération IA (préparation 3.2)

- **Cohérence par prompt-système** : chaque prompt de la bibliothèque embarquera un « bloc DA » commun (palette, lumière, ambiance, niveau de réalisme) pour garantir l'unité entre générations.
- **Pièges à éviter** : anatomie animale incorrecte (membres, yeux), produits flottants/incohérents entre vues, textes générés illisibles (le texte est toujours ajouté en post-prod, jamais généré), sur-stylisation « IA lisse » incompatible avec le réalisme premium.
- **Negative prompts systématiques** : déformations anatomiques, watermark, texte, mains humaines mal rendues, sursaturation.
- Les visuels où le produit exact compte (packshots des vrais produits vendus) devront à terme être remplacés par de vraies photos — les générations IA servent de **placeholders haute qualité et de direction artistique** (**H32**).

## 5. Erreurs fréquentes à éviter

1. Mélanger des styles d'images disparates (stock + IA + photo) sans DA commune — l'incohérence se voit immédiatement.
2. Hero sans espace négatif → texte illisible, contraste AA impossible (spec 2.1 Accueil).
3. Packshots aux fonds légèrement différents d'un produit à l'autre (effet marketplace).
4. Images lourdes non optimisées — la moitié du LCP se joue là.
5. Animaux anthropomorphisés/costumés — contraire au ton D-008.
