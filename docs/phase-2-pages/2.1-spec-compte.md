# Phase 2.1 — Spécification de page : Compte client

> **Statut** : 🟡 En attente de validation HITL
> **URLs** : `/compte/*` (voir Sitemap 1.2 §2.4) + `/compte/connexion`, `/suivi-commande` (invité)
> **Équipe** : UX/UI Designer, Product Owner, QA, Copywriter
> **S'appuie sur** : recherche 2.0 Compte (validée), D-014/D-015/D-016, objectif réachat D-009.

---

## 1. Objectif

Faire revenir et racheter : chaque écran doit rendre un service concret (suivre, racheter, retourner, personnaliser). **KPIs** : taux de réachat ≥ 25 % à 6 mois, ≥ 50 % des clients avec ≥ 1 animal renseigné, ≥ 70 % des retours déclarés en self-service.

## 2. Authentification

### `/compte/connexion`
- Onglets `Connexion` / `Créer un compte` ; e-mail + mot de passe ; lien `Mot de passe oublié` → `/compte/mot-de-passe` (e-mail de réinitialisation, lien 1 h).
- Création : e-mail + mot de passe (indicateur de robustesse, min. 8 caractères) ; pas de double saisie (afficher/masquer à la place).
- Rappel : la création est aussi proposée post-achat en 1 clic (D-014) — même compte, même flux d'onboarding animal.
- Redirection intelligente : retour à la page d'origine après connexion (panier conservé, fusion D-030).
- **H23** : pas de connexion sociale (Google/Apple) au lancement — simplicité ; à réévaluer.

## 3. Layout

- **Desktop** : navigation latérale gauche persistante ; contenu à droite.
- **Mobile** : `/compte` = liste de sections (cards) ; navigation par pile avec retour.
- Menu (7 chemins, recherche 2.0) : `Tableau de bord` · `Mes commandes` · `Mes animaux` · `Mes adresses` · `Mes informations` · `Préférences e-mail` · `Aide & retours` + `Déconnexion`.

## 4. Écrans

### 4.1 Tableau de bord (`/compte`)
1. Salutation : `Bonjour {prénom}` (+ `et Max 🐾` si animal renseigné — unique emoji toléré, à trancher en DA Phase 4).
2. **Commande en cours** (si active) : carte statut (D-016) + suivi + `Voir le détail`. Sinon masqué.
3. **Racheter facilement** : 2–4 produits des dernières commandes avec ajout direct.
4. **Mes animaux** : cartes animaux ou invitation à créer (`Ajoutez votre compagnon pour des recommandations à sa taille`).
5. **Sélection pour {animal}** : 4 recommandations personnalisées (masqué sans profil).

### 4.2 Mes commandes (`/compte/commandes`, `/compte/commandes/{id}`)
- Liste antichronologique : cartes avec vignettes produits, date, total, statut coloré, actions rapides (`Suivre`, `Racheter`, `Facture PDF`).
- Détail : timeline de statut (D-016), articles (lien fiche), adresse, paiement (`•••• 4242`), totaux, bouton `Déclarer un retour` (si ≤ 30 j après livraison).
- **Flux retour self-service** : sélection articles → motif (liste + champ libre) → confirmation → étiquette PDF/e-mail → suivi du remboursement dans la timeline. ≤ 3 écrans.
- **Racheter** : re-ajout panier avec vérification stock ; les indisponibles sont signalés (`2 articles sur 3 ajoutés`).

### 4.3 Mes animaux (`/compte/animaux`) — section signature
- Cartes par animal : nom, photo (optionnelle), espèce, race (autocomplétion, facultative), gabarit (XS→XL, aligné sur le filtre D-027), date de naissance approximative (mois/année).
- CRUD complet ; suppression avec confirmation douce.
- **Bandeau bénéfice** : `Les tailles recommandées pour {nom} s'affichent désormais sur les fiches produit.` — le service rendu est explicite (recherche 2.0).
- Limite raisonnable : 5 animaux (**H24**).

### 4.4 Mes adresses (`/compte/adresses`)
Cartes adresse (livraison par défaut marquée) ; ajout/édition via `AddressForm` (partagé checkout, D-032) ; suppression protégée si commande en cours.

### 4.5 Mes informations (`/compte/informations`)
- Prénom, nom, e-mail, mot de passe — **édition explicite champ par champ** (modale ou inline avec confirmation), changement d'e-mail ou de mot de passe = confirmation par e-mail + re-authentification.
- **RGPD self-service** : `Télécharger mes données` (export JSON/CSV asynchrone par e-mail) ; `Supprimer mon compte` (confirmation forte, mention des commandes conservées pour obligations légales, anonymisation — cohérent User Flows 3.3).

### 4.6 Préférences e-mail (`/compte/preferences`)
Toggles : `Newsletter mensuelle` · `Conseils personnalisés` · (transactionnel non désactivable, listé grisé pour transparence). Lien de désinscription des e-mails → cette page, pré-authentifiée par token.

### 4.7 Aide & retours (`/compte/aide`)
FAQ ciblée client connecté + contact pré-rempli (commande sélectionnable) + rappel promesse `< 24 h ouvrées`.

### 4.8 Suivi invité (`/suivi-commande`)
N° de commande + e-mail → page de suivi en lecture (timeline, articles) + `Déclarer un retour` (même flux self-service, authentifié par le couple n°/e-mail). Bandeau discret : `Créez un compte pour retrouver toutes vos commandes`.

## 5. États & cas limites (QA)

- Nouveau compte sans commande : dashboard = invitation animaux + best-sellers (jamais vide).
- Commande annulée/remboursée : statuts et montants clairs dans la timeline.
- Retour hors délai : bouton remplacé par un lien contact explicatif.
- Session expirée : reconnexion → retour à l'écran demandé.
- Compte supprimé : e-mail de confirmation, déconnexion immédiate, panier invité conservé.

## 7. Sécurité (préfigure Phase 6)

Sessions httpOnly ; re-authentification pour données sensibles ; rate-limiting sur connexion/réinitialisation ; messages d'erreur non énumérants (`E-mail ou mot de passe incorrect`) ; tokens de réinitialisation à usage unique.

## 8. Hypothèses

- **H23** : pas de connexion sociale au lancement.
- **H24** : maximum 5 animaux par compte.
- **H25** : la photo d'animal est stockée mais jamais affichée publiquement (usage compte uniquement).
