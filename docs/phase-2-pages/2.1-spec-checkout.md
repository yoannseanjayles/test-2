# Phase 2.1 — Spécification de page : Checkout

> **Statut** : 🟡 En attente de validation HITL
> **URLs** : `/checkout` (étapes en état interne, une seule URL) · `/checkout/confirmation/{token}`
> **Équipe** : UX/UI Designer, Architecte Front-end, QA, Copywriter
> **S'appuie sur** : recherche 2.0 Checkout (validée), D-004, D-014, D-029/D-030, H5, H9, H18.

---

## 1. Objectif

Faire aboutir un panier en < 3 minutes, sans surprise ni perte de données. **KPIs** : abandon checkout ≤ 55 % (D-009), complétion mobile ≥ complétion desktop − 5 pts, taux d'erreur de paiement < 3 %.

## 2. Layout

- **Header épuré** : logo (cliquable, avec garde-fou `Votre panier est sauvegardé`), réassurance (`Paiement sécurisé · Retours 30 jours`), pas de navigation ni recherche.
- **Desktop** : formulaire à gauche (60 %), `OrderSummary` sticky à droite (40 %) — articles, sous-total, livraison, total TTC, code promo replié (D-030).
- **Mobile** : récap en accordéon replié en haut (`Voir le détail — 84,90 €`), formulaire 1 colonne, CTA pleine largeur.
- **Stepper visuel** : `1. Coordonnées → 2. Livraison → 3. Paiement` ; étapes validées affichées en résumé compact avec lien `Modifier` (édition en place, retour automatique à l'étape courante — linéarité stricte).

## 3. Étapes

### Étape 0 — Express (au-dessus de tout)
`ExpressPaymentButtons` : Apple Pay · Google Pay · PayPal, séparateur `ou continuer avec votre e-mail`. Les wallets fournissent adresse + paiement → sautent aux frais de livraison puis confirmation.

### Étape 1 — Coordonnées
- **E-mail** (autocomplete `email`, validation format + suggestion de typo `gmial.com → gmail.com ?`).
- Si l'e-mail correspond à un compte : proposition de connexion **non bloquante** (`Ravi de vous revoir — connectez-vous ou continuez en invité`).
- Opt-in newsletter décoché par défaut (RGPD).
- Dès validation : le panier est associé à l'e-mail (relance H9).

### Étape 2 — Livraison
- **7 champs** : prénom, nom, adresse (autocomplétion BAN/API + autofill navigateur), complément (replié sous `+ Ajouter un complément`), code postal → ville pré-remplie éditable, pays (FR défaut ; BE/CH/LU — H5), téléphone optionnel avec justification (`Uniquement pour la livraison`).
- **Modes de livraison** : cartes radio avec prix et délai réels (`Colissimo domicile — 4,90 € — 2/3 j ouvrés` · `Point relais — 3,90 €` · `Express — 9,90 € — 24 h`), gratuité appliquée au-delà du seuil (D-029). Point relais → sélecteur carte/liste en overlay.
- Client connecté : adresses enregistrées en cartes sélectionnables + `Nouvelle adresse`.

### Étape 3 — Paiement
- **Carte** (défaut) : numéro (auto-format espaces, Luhn, détection réseau, `inputmode=numeric`), expiration `MM/AA`, CVC (aide visuelle), nom du titulaire. Adresse de facturation = livraison par défaut (case décochable).
- PayPal en alternative radio (redirection annoncée).
- Note 3DS : `Votre banque peut vous demander une confirmation (3-D Secure).`
- **CTA final** : `Payer 84,90 €` (montant dynamique) + badges + `Retours 30 jours` adjacents.
- CGV : mention `En payant, vous acceptez nos CGV` avec lien (pas de case à cocher supplémentaire — friction inutile, standard FR accepté).

### Confirmation (`/checkout/confirmation/{token}`)
1. `Merci ! Votre commande n°{ref} est confirmée.` + e-mail de confirmation envoyé.
2. Récap : articles, adresse, mode + délai estimé, total.
3. **Création de compte 1 clic** : champ mot de passe seul + bénéfices (`Suivez votre commande, gérez vos retours`) → puis invitation profil animal (D-015), ignorable.
4. Rebond doux : 2–3 guides recommandés (pas de vente agressive post-achat).

## 4. Règles de validation & erreurs (QA)

- Validation **à la sortie du champ** ; jamais pendant la frappe ; messages spécifiques (`Le code postal doit comporter 5 chiffres`), champ + message liés par `aria-describedby`.
- Échec de soumission : **toutes les données conservées**, autoscroll + focus sur la première erreur.
- **Échec de paiement** : retour à l'étape 3 avec message précis non technique (`Paiement refusé par votre banque — essayez une autre carte`), panier et champs intacts (carte à re-saisir : PCI), alternatives suggérées.
- Session expirée / stock épuisé pendant le checkout : bandeau explicite + retour panier avec état préservé.
- Double-clic sur `Payer` : bouton verrouillé + spinner (idempotence côté back, Phase 6).
- Prix/livraison recalculés côté serveur à chaque étape — jamais de confiance au front (sécurité, Phase 6).

## 5. Accessibilité & performance

- Formulaire entièrement navigable au clavier ; stepper annoncé (`étape 2 sur 3`) ; erreurs annoncées `aria-live=assertive` ; contrastes AA.
- Attributs `autocomplete` complets (`email`, `given-name`, `family-name`, `street-address`, `postal-code`, `tel`, `cc-number`…) — l'autofill est le premier accélérateur de complétion.
- Page légère : aucun script non essentiel au tunnel, champs de carte en iframe PSP (PCI-DSS, Phase 6).
