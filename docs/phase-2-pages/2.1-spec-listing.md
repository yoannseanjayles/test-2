# Phase 2.1 — Spécification de page : Listing / pages catégorie

> **Statut** : 🟡 En attente de validation HITL
> **URLs** : `/{animal}` (gabarit A — aperçu) · `/{animal}/{sous-categorie}` (gabarit B — listing) · `/nouveautes` et `/recherche` (variantes du gabarit B)
> **Équipe** : UX/UI Designer, Expert SEO, Architecte Front-end, QA
> **S'appuie sur** : recherche 2.0 Listing (validée), D-002, D-012, D-013, D-024.

---

## Gabarit A — Page animal (`/chien`, `/chat`, `/nac`)

### Objectif
Router vers la bonne sous-catégorie en donnant une vue d'ensemble de l'univers (D-002 : catégorie parente cliquable = vraie page).

### Structure
1. **En-tête d'univers** : H1 (`Accessoires pour chien`), texte de curation court (2 phrases), visuel de bandeau discret.
2. **Grille de sous-catégories** : cartes `CategoryCard` (photo + libellé + nb de produits) — les 8 sous-catégories chien (D-012).
3. **Best-sellers de l'univers** : 4–8 `ProductCard`, CTA `Voir tout`.
4. **Guides de l'univers** : 2–3 `EditorialCard` filtrées par espèce.
5. **Réassurance + newsletter + footer** (composants partagés).

### KPI : ≥ 70 % de clics vers une sous-catégorie ou un produit.

## Gabarit B — Page sous-catégorie (`/chien/colliers-harnais`)

### Objectif
Permettre de comparer et choisir : trouver le bon produit en < 3 interactions (1 filtre + 1 tri + 1 clic).

**KPIs** : CTR vers fiche ≥ 40 %, usage des filtres ≥ 30 %, zéro état vide sans issue.

### Structure

#### S1. En-tête
- Fil d'Ariane ; H1 (`Colliers & Harnais pour chien`) + description SEO/curation 2 phrases (`Chaque collier est testé et validé par notre équipe.`) ; nombre de produits.

#### S2. Barre d'outils
- **Desktop** : facettes en colonne gauche sticky ; barre supérieure = filtres appliqués (chips supprimables + `Tout effacer`) + tri à droite.
- **Mobile** : 2 boutons sticky sous le header — `Filtrer (2)` et `Trier` ; panneau plein écran avec compteur live (`Voir 24 produits`) et `Réinitialiser`.
- **Tris** : `Notre sélection` (défaut) · `Prix ↑` · `Prix ↓` · `Meilleures notes` · `Nouveautés`.
- **Bouton « Filtrer pour {animal} »** (connecté avec profil) : applique gabarit + taille en 1 clic, retirable comme tout chip.

#### S3. Facettes
| Facette | Type | Portée |
|---|---|---|
| Gabarit animal (`XS < 5 kg` → `XL > 40 kg`) | Cases à cocher | **Toutes sous-catégories** (signature) |
| Taille produit (S/M/L…) | Cases | Toutes |
| Prix | Curseur min–max + champs | Toutes |
| Matière | Cases | Toutes |
| Couleur | Pastilles | Toutes |
| Marque | Cases + recherche si > 10 | Toutes |
| Spécifiques (ex. `Tour de cou`, `Rembourrage`, `Réfléchissant`) | Selon attributs | Par sous-catégorie (config Phase 6) |

Règles : application immédiate (desktop), valeurs ordonnées logiquement, compteur de résultats par valeur, combinaisons vides désactivées (grisées), multi-sélection = OR dans une facette, AND entre facettes.

#### S4. Grille produits
- 4 colonnes desktop / 2 mobile ; ~24 produits par lot ; bouton **`Charger plus`** + indicateur de position (`24 sur 87`) ; position mémorisée au retour depuis une fiche.
- `ProductCard` (validée) : photo (lifestyle au hover), nom, prix, note + nb avis, pastilles couleurs, tailles dispo, badge `Nouveau`. Rupture totale : carte grisée `Bientôt de retour` en fin de grille.
- **Carte éditoriale insérée** en position 6–8 si un guide correspond à la sous-catégorie (1 max).

#### S5. Bas de page
- Bloc SEO dépliable (texte catégorie long, maillage vers guides et sous-catégories sœurs) ; footer.

### États & cas limites (QA)
- **0 résultat après filtre** : message + suggestion de retirer le dernier filtre appliqué (chip mis en évidence) + best-sellers de la catégorie en secours.
- **Catégorie < 10 produits actifs** : alerte back-office (D-002) ; côté front, la grille masque le compteur de facettes inutiles.
- **Chargement** : squelettes de cartes (pas de spinner global) ; les filtres restent utilisables.
- **URL partagée avec filtres** : l'état complet (filtres + tri) est restauré depuis la query-string.

### Variantes du gabarit B
- **`/nouveautes`** : pas de facettes spécifiques, facette `Univers` (Chien/Chat/NAC) en plus, tri par défaut `Nouveautés`.
- **`/recherche?q=`** : en-tête = `Résultats pour « harnais » (32)` ; onglets `Produits / Guides` ; 0 résultat → suggestions de catégories populaires + guides (User Flows 1.2). Non indexée.

### SEO
- Title `Colliers & harnais pour chien — sélection premium | {Marque}` ; meta description par catégorie.
- Données structurées `ItemList` + `BreadcrumbList`.
- Filtres en query-string, **canonique vers la sous-catégorie nue** ; pagination « charger plus » doublée de vraies URLs `?page=n` accessibles aux crawlers (progressive enhancement), canonique self + maillage.
- Pages d'atterrissage éditorialisées pour les requêtes à fort volume : hors périmètre du lancement, prévues en évolution (**H16**).
