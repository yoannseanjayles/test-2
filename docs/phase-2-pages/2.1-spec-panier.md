## A. Mini-panier (drawer)

### Objectif
Confirmer l'ajout sans rupture de contexte, pousser vers le seuil de livraison offerte, offrir une sortie rapide vers le checkout.

**KPIs** : taux de passage drawer → checkout ≥ 35 %, contribution à l'AOV (suivi du taux de paniers ≥ 79 €).

### Comportement
- S'ouvre en slide-in droit (400 ms max) à chaque ajout ; fermeture par ×, overlay, Échap. Accessible : focus-trap, `aria-live` sur l'ajout.
- Icône panier du header : badge quantité mis à jour en temps réel.

### Structure (haut → bas)
1. **Titre** `Votre panier (3)` + ×.
2. **FreeShippingBar** : `Plus que 12 € pour la livraison offerte` + barre de progression ; état atteint : `🎉 Livraison offerte` (vert).
3. **Lignes articles** (compactes) : vignette, nom, variante `Taille M · Sable`, contexte animal éventuel (`Pour Max`), stepper quantité, prix ligne, suppression (undo toast 5 s).
4. **Cross-sell curé** : `Complétez` — 1–2 produits liés au dernier ajout, carte mini + ajout 1-clic. Masqué si le panier contient ≥ 4 articles (ne pas retarder la caisse).
5. **Pied sticky** : sous-total + estimation livraison (`Livraison estimée : 4,90 € — offerte dès 79 €`), CTA primaire `Commander`, lien tertiaire `Voir le panier`.

## B. Page panier (`/panier`)

### Objectif
Vérifier, ajuster, rassurer : zéro surprise de coût, transition sereine vers le checkout.

**KPIs** : passage panier → checkout ≥ 60 %, abandon checkout global ≤ 55 % (D-009).

### Structure

#### Desktop : 2 colonnes (articles 60 % / récap sticky 40 %) · Mobile : articles puis récap, CTA sticky bas

#### S1. En-tête
`Votre panier (3 articles)` + FreeShippingBar.

#### S2. Lignes articles
- Vignette cliquable, nom + lien fiche, **variante modifiable en place** (select taille/couleur inline, sans supprimer), contexte animal (`Recommandé pour Max — taille M`), stepper quantité, prix unitaire + total ligne, `Expédié sous 24 h` par article, suppression avec undo.
- Article devenu indisponible : ligne grisée + alternatives proposées, non comptée dans le total.

#### S3. Récapitulatif (sticky)
1. Sous-total ; **Livraison estimée** (FR par défaut, sélecteur pays FR/BE/CH/LU — H5) ; Total TTC (`TVA incluse`).
2. CTA primaire `Passer commande`.
3. **Paiements express** : Apple Pay · Google Pay · PayPal (séparés par `ou`).
4. Lien replié `Vous avez un code ?` → champ + validation inline (erreur douce, jamais bloquante).
5. Réassurance : `Paiement sécurisé` · `Retours 30 jours` · logos moyens de paiement.

#### S4. Sous le pli
`Vous aimerez aussi` (4 ProductCard max, curées) ; maillage continuité (retour catégorie).

### États & cas limites (QA)
- **Panier vide** : illustration sobre + `Votre panier est vide` + CTA vers les 3 univers + best-sellers. Jamais de cul-de-sac.
- **Panier persistant** : invité = cookie 30 jours ; connecté = fusion serveur (le panier invité rejoint le compte à la connexion, avec dédoublonnage quantités).
- **Changement de prix/stock depuis l'ajout** : bandeau informatif explicite (`Le prix de X a changé`), jamais de modification silencieuse.
- **Quantité max** : plafond par stock disponible, message inline.
- **Code promo invalide/expiré** : message précis (`Ce code a expiré le…`), champ conservé.

### SEO
`noindex` ; aucun besoin de données structurées.
