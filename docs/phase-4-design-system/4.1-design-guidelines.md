## 1. Direction artistique

### Nom de marque (résolution de H4)

**Proposition : élever le nom de code « Pelage » au rang de nom de marque.**
Justification : français, un seul mot, évocateur sans être infantile, disponible en signature sobre (`Pelage — accessoires d'exception`), fonctionne en logotype typographique. *Alternative si refusé : garder « Pelage » comme code et trancher le naming avec un exercice dédié.*

### Principes visuels (découlent de D-008/D-044)

1. **Quiet luxury** : la qualité se montre, elle ne se crie pas — espaces généreux, peu d'éléments par écran, aucune agressivité commerciale (pas de compte à rebours, pas de rouge promo).
2. **Chaleur naturelle** : palette issue des matières réelles (cuir, lin, laine, bois) — le site ressemble aux produits.
3. **L'animal d'abord** : la photographie porte l'émotion ; l'UI s'efface derrière elle (fonds neutres, chrome discret).
4. **Précision artisanale** : alignements exacts, rythme vertical régulier, détails soignés (le « bien fini » se ressent).

## 4. Grille & espacements

- **Espacement base 4** : `space-0.5`(2) `1`(4) `2`(8) `3`(12) `4`(16) `6`(24) `8`(32) `12`(48) `16`(64) `24`(96) px.
- **Grille** : 12 col desktop, gouttière `space-6` ; 4 col mobile, gouttière `space-4` ; conteneur max **1360 px** + marges `space-6` (mobile `space-4`).
- **Breakpoints** : `sm 640` · `md 768` · `lg 1024` · `xl 1280` (mobile-first).
- **Rayons** : `radius-sm 6` (inputs, chips) · `radius-md 10` (cartes, boutons) · `radius-lg 16` (drawers, modales, images) · `radius-full` (pastilles). Doux mais pas « bulle ».
- **Ombres** : 2 niveaux seulement — `shadow-card` (0 1px 3px rgba(43,38,32,.08)) et `shadow-overlay` (0 8px 30px rgba(43,38,32,.16)). Le premium préfère les filets aux ombres fortes.
- Rythme vertical : sections de page espacées de `space-16` desktop / `space-12` mobile.

## 5. Iconographie

- Style : **ligne 1.75px, bouts ronds**, grille 24px (cohérent M-ICO, D-043), coin arrondi 2px ; taille d'usage 20/24px, `color-text-secondary` par défaut.
- Bibliothèque de base : **Lucide** (open source, cohérente avec ce style) + icônes custom espèces/piliers (M-ICO-01…10) dessinées sur la même grille (**H35**).
- Toujours accompagnées de texte ou `aria-label` — jamais d'icône seule porteuse de sens (D-045).

## 6. Composants — standards transverses

*(Le registre détaillé vient des specs Phase 2 ; ici : les règles communes.)*

### Boutons
| Variante | Style | Usage (hiérarchie D-022) |
|---|---|---|
| `primary` | Fond `color-action`, texte blanc, radius-md, padding 12×24 | 1 par écran max |
| `secondary` | Contour 1.5px `color-action`, texte `color-action` | Actions secondaires |
| `tertiary` | Texte `color-action` + flèche, sans fond | Liens d'action |
| `ghost` | Texte `bark-700`, hover fond `cream-300` | Navigation, outils |

États obligatoires : default / hover (assombrissement + translation −1px subtile) / focus (`focus-ring`) / active / disabled (opacité .45, curseur) / **loading (spinner inline, label conservé)**. Hauteur min 44px.

### Formulaires (FormField — D-032)
Label au-dessus (`label`), aide en dessous (`body-small`, `bark-700`), erreur (`error` + icône + `aria-describedby`), input hauteur 48px, fond `cream-50`, bordure `color-border` → `pine-500` au focus. Validation à la sortie du champ (D-033).

### Cartes (ProductCard & dérivées)
Fond `cream-50`, radius-lg sur l'image, pas d'ombre au repos (filet `color-border`), hover : `shadow-card` + image lifestyle (PDP hover, D-027) ; toute la carte cliquable avec focus visible englobant.

### Overlays (drawer, modale, overlay guide tailles)
Fond `cream-50`, radius-lg côté ouvert, voile `rgba(43,38,32,.4)`, focus-trap, fermeture Échap/voile/× (specs 2.1), largeur drawer 440px desktop / 100% mobile.

### Badges & chips
`Nouveau` : fond `sage-100`, texte `sage-700` · rupture : `cream-300`/`bark-700` · FilterChip actif : fond `pine-700`, blanc, × intégré.

## 7. Responsive — règles système

- Mobile-first ; les layouts 2 colonnes (PDP, panier, checkout) basculent à `lg`.
- Navigation : barre fixe basse mobile (Sitemap 1.2), méga-menu ≥ `lg` seulement.
- Images : ratios réservés via CSS `aspect-ratio` (zéro layout shift — CLS, D-009).
- Zones tactiles ≥ 44×44 ; steppers et facettes espacés `space-2` min.

## 8. Motion & micro-interactions

| Token | Valeur | Usage |
|---|---|---|
| `motion-fast` | 150ms ease-out | Hover, toggles, chips |
| `motion-base` | 250ms ease-out | Accordéons, fades, cartes |
| `motion-slow` | 400ms cubic-bezier(.22,1,.36,1) | Drawers, modales, méga-menu |

- Uniquement `transform` + `opacity` (perf) ; ampleur ≤ 16px.
- Signatures : fade-up d'entrée de sections (une fois, 250ms), slide-in drawer (400ms), pulse discret du badge panier à l'ajout, progression FreeShippingBar animée, `Ajouté ✓` sur CTA (1.5s).
- **`prefers-reduced-motion: reduce`** → toutes les transitions ramenées à 0ms (règle système, non négociable).
- Pas de parallaxe, pas d'animations en boucle, pas d'auto-play (D-020).

## 9. Voix & ton (rappel Copywriter — D-008)

Chaleureux, précis, sobre. Vouvoiement. Verbes d'action sur les CTA. Jamais de « toutou/minou », jamais d'urgence artificielle. L'animal est nommé par son prénom quand il est connu (D-023).

## 10. Accessibilité — synthèse des règles système

Contrastes AA validés par paires de tokens (§2) · focus visible universel (`focus-ring`) · navigation clavier complète · cibles 44px · `aria-live` sur les changements dynamiques (panier, erreurs) · formulaires liés (`label`/`aria-describedby`) · reduced-motion · jamais de couleur seule porteuse d'information · HTML sémantique (landmarks, headings hiérarchiques — Phase 5).

## Hypothèses

- **H35** : icônes custom (espèces, piliers) dessinées sur la grille Lucide plutôt que générées — cohérence garantie.
- **H36** : pas de dark mode au lancement (la palette crème est l'identité) ; l'architecture de tokens le permet plus tard sans refonte.
