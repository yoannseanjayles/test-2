# Phase 1.0 — Best Practices Summary

## Architecture, navigation, sitemap et parcours utilisateurs — E-commerce premium d'accessoires pour animaux

> **Statut** : 🟡 En attente de validation HITL
> **Équipe** : Product Manager, UX Researcher, Architecte Front-end, Expert SEO
> **Sources principales** : Baymard Institute (200 000+ h de recherche UX), benchmarks des leaders du marché pet (Chewy, Zooplus, Petco), guides d'architecture e-commerce 2026.

---

## 1. Architecture e-commerce

### Principes retenus

| Principe | Détail | Pourquoi c'est pertinent pour nous |
|---|---|---|
| **Hiérarchie plate (règle des 3 clics)** | Modèle `Accueil → Catégorie → Sous-catégorie → Produit`. Aucune page stratégique à plus de 3 niveaux de profondeur. | Un catalogue d'accessoires (colliers, laisses, couchages, jouets…) se structure naturellement en 2 niveaux ; inutile de complexifier. |
| **Taxonomie double : par animal ET par usage** | Les leaders (Chewy, Zooplus) croisent deux entrées : *espèce* (chien, chat, NAC) et *type de produit* (promenade, couchage, jeu…). | Le visiteur pense d'abord « pour mon chien », puis « une laisse ». L'entrée par animal doit être primaire, l'entrée par usage secondaire (facettes/sous-menus). |
| **Séparation contenu / catalogue** | Le contenu éditorial (guides, blog) vit dans une arborescence propre mais maille vers le catalogue. | Positionnement premium = conseil et expertise. Le contenu alimente le SEO et nourrit la considération. |
| **Architecture évolutive** | Prévoir dès le sitemap les emplacements futurs (marques, collections, cadeaux) sans les bloquer. | Exigence du brief : produit repris facilement par n'importe quelle équipe. |

### Règles de taxonomie (Baymard)

- **Catégories parentes cliquables** : un intitulé de menu parent doit mener à une vraie page d'aperçu de la catégorie, pas être un simple label — les utilisateurs s'en servent pour se faire une vue d'ensemble avant de choisir une sous-catégorie.
- **Taille des catégories** : subdiviser à partir de ~10 sous-catégories ; viser **au moins 10 produits** dans chaque catégorie feuille. Une catégorie avec 2 produits déçoit et casse la perception premium.
- **Pas de redondance** : éviter les catégories croisées qui dupliquent les produits sans logique (source de confusion + contenu dupliqué SEO).
- **Révision périodique** : la taxonomie doit être auditée régulièrement (métier + data).

## 2. Navigation

### Constats de la recherche

- Jusqu'à **67 % des sites leaders** ont une navigation jugée « médiocre à mauvaise » (benchmark Baymard) — bien exécuter ce point est un différenciateur réel.
- **69 % des utilisateurs** utilisent la navigation ou la recherche comme méthode principale pour trouver un produit ; une navigation optimisée peut augmenter la conversion de 30 à 60 %.
- **43 % des acheteurs** vont directement à la barre de recherche et convertissent 2 à 3× plus : la recherche interne est un canal de conversion, pas un gadget.

### Principes retenus

1. **Méga-menu structuré par animal** : entrées de premier niveau = Chien / Chat / NAC (+ entrées transverses : Nouveautés, Guides, Marque). Le méga-menu expose les sous-catégories avec visuels.
2. **Mobile-first réel** : ~73 % du revenu pet e-commerce se fait sur mobile. Les catégories produits doivent être les items de premier niveau du menu mobile (33 % des sites l'oublient — erreur fréquente identifiée par Baymard).
3. **Fil d'Ariane systématique** sur catégorie, sous-catégorie et fiche produit (orientation + SEO).
4. **Recherche proéminente** avec autocomplétion (produits, catégories, contenus) et tolérance aux fautes.
5. **Navigation à facettes maîtrisée** : filtres utiles (taille de l'animal, matière, couleur, prix, marque) ; côté SEO, décider explicitement quelles pages filtrées sont indexables pour éviter la prolifération d'URLs — c'est le principal risque SEO d'un e-commerce.
6. **Footer riche** : réassurance (livraison, retours, contact), accès secondaires (CGV, FAQ, blog), newsletter.

## 3. Sitemap — bonnes pratiques

- **Pages indispensables identifiées** (seront détaillées en 1.2) : accueil, pages catégories/sous-catégories, fiche produit, recherche, panier, checkout, compte client, suivi de commande, contenu éditorial (guides/blog), pages de confiance (à propos, livraison & retours, FAQ, contact), pages légales.
- **URLs propres et hiérarchiques** : `/chien/colliers/collier-cuir-x` — lisibles, stables, reflétant la taxonomie.
- **Sitemap XML** segmenté (produits / catégories / contenus) + maillage interne cohérent ; les pages orphelines sont proscrites.
- **Erreur fréquente à éviter** : construire le sitemap depuis l'organisation interne (« nos gammes fournisseurs ») plutôt que depuis le modèle mental du client (« mon animal, mon besoin »).

## 4. Parcours utilisateurs

### Visiteur → Client (achat)

- **Entrées multiples** : la majorité des sessions n'arrivent pas par la home (SEO catégorie/produit, social, ads). Chaque page doit être un point d'entrée autonome : contexte, réassurance et navigation présents partout.
- **Checkout = zone critique** : ~70 % des paniers sont abandonnés, majoritairement pour des raisons UX. Standards à intégrer dès la conception :
  - **checkout invité** (24 % abandonnent si la création de compte est forcée) — le compte se propose *après* l'achat ;
  - **transparence des coûts** dès le panier (frais de port surprises = 1ʳᵉ cause d'abandon, ~40 %) ;
  - **paiements diversifiés** (CB, Apple Pay / Google Pay, PayPal ; +12–15 % de conversion) et express checkout ;
  - **formulaires courts**, une colonne sur mobile, indicateur d'étapes ;
  - **signaux de confiance adjacents au paiement** (badges sécurité, rappel retours : +18 % de complétion mesurée).

### Client fidèle (spécificité du marché pet)

- Le marché pet est un marché de **réachat récurrent** : chez Chewy, ~70 % du CA provient de l'abonnement Autoship.
- Parcours à prévoir : **« racheter facilement »** (historique, buy-again), profil animal (espèce, taille, âge) pour personnaliser les recommandations, et — en évolution — abonnement/réassort pour les consommables.
- Le **profil de l'animal** est le levier de personnalisation différenciant du secteur (recommandations par taille/race/âge) — c'est un parcours d'onboarding à part entière.

### Administrateur

- Parcours back-office standards : gestion catalogue (produits, variantes, stocks), commandes (statuts, remboursements), clients, contenus, promotions.
- Bonne pratique : concevoir les parcours admin **en miroir des parcours client** (une commande côté client = un flux de traitement côté admin) — détaillé en Phase 7.

## 5. Synthèse — ce que nous appliquerons

1. Taxonomie **primaire par animal** (Chien / Chat / NAC), secondaire par usage, catégories parentes cliquables, ≥10 produits par catégorie feuille.
2. Architecture plate ≤ 3 niveaux, URLs hiérarchiques propres, contenu éditorial séparé mais maillé.
3. Navigation mobile-first : catégories en premier niveau, méga-menu desktop, fil d'Ariane, recherche avec autocomplétion.
4. Parcours d'achat sans friction : checkout invité, coûts transparents, paiements express, réassurance au paiement.
5. Parcours de fidélisation natifs : profil animal, buy-again, socle prêt pour l'abonnement.
6. Facettes utiles côté UX, indexation maîtrisée côté SEO.

### Hypothèses signalées

- **H1** : la cible NAC est secondaire en volume par rapport à chien/chat — la taxonomie NAC sera moins profonde au lancement.
- **H2** : le catalogue initial est composé d'accessoires (pas d'alimentation) — l'abonnement/réassort est donc prévu comme évolution, pas comme fonctionnalité de lancement.
- **H3** : marché francophone au lancement (langue FR, EUR) — l'i18n reste une évolution possible.
